### 2-3. 코드 작성

**파일**: `exercises/elementary_1.1_code_examples.mlir` (254줄)

**작성한 코드**:

#### ✅ 올바른 예제 (5개)

1. **예제 1: 기본 더하기**
   ```mlir
   %0 = arith.addi %a, %b : i32
   return %0 : i32
   ```
   - %0이 정확히 한 번만 정의 ✅
   - SSA 규칙 준수 ✅

2. **예제 2: 연쇄 계산**
   ```mlir
   %0 = arith.addi %a, %b : i32
   %1 = arith.muli %0, 2 : i32
   %2 = arith.subi %1, 3 : i32
   ```
   - %0, %1, %2 각각 한 번씩만 정의 ✅
   - 완벽한 SSA 형식 ✅

3. **예제 3: 여러 입력값**
   ```mlir
   %0 = arith.addi %apple, %banana : i32
   %1 = arith.addi %0, %orange : i32
   ```
   - 함수 인자는 여러 번 사용 가능 ✅
   - 새로운 이름표(%0, %1) 각각 한 번씩 ✅

4. **예제 4: 의미 있는 이름**
   ```mlir
   %sum = arith.addf %x, %y : f32
   %doubled = arith.mulf %sum, 2.0 : f32
   %result = arith.subf %doubled, 1.0 : f32
   ```
   - 의미 있는 이름표 사용 ✅
   - 각 이름표 한 번씩만 정의 ✅

5. **예제 5: 상수 사용**
   ```mlir
   %0 = arith.constant 10 : i32
   %1 = arith.addi %n, %0 : i32
   %2 = arith.muli %1, 2 : i32
   ```
   - 상수도 SSA 값으로 처리 ✅
   - 완전한 SSA 형식 ✅

#### ❌ 의도적 오류 예제 (2개)

1. **틀린 예제 1: 같은 이름표 두 번**
   ```mlir
   %0 = arith.addi %a, %b : i32
   %0 = arith.muli %0, 2 : i32  // ❌ %0을 다시 정의!
   ```
   - SSA 규칙 위반 명확 ✅
   - 왜 틀렸는지 설명함 ✅

2. **틀린 예제 2: 재정의**
   ```mlir
   %sum = arith.addi %x, %y : i32
   %sum = arith.addi %sum, %z : i32  // ❌ %sum을 다시 정의!
   ```
   - 의미 있는 이름 사용해도 SSA 규칙 위반 ✅
   - 명확한 에러 설명 ✅

#### ✅ 수정된 예제 (1개)

```mlir
%0 = arith.addi %x, %y : i32
%1 = arith.addi %0, %z : i32
return %1 : i32
```
- 틀린 예제 2의 올바른 수정 ✅

#### ✅ 실습 문제 (3개)

- **문제 A** (올바름): %a, %b 다른 이름표
- **문제 B** (틀림): %result 두 번 정의
- **문제 C** (올바름): %0, %1, %2 모두 다른 이름표
