## 핵심 개념 정리

### 개념 1: SSA (Static Single Assignment)

**정의 (쉬운 말로)**:
- 모든 결과값의 이름표는 정확히 한 번만 정의됨
- 이름표를 다시 쓸 수 없음

**예시**:
```
✅ 올바름: %0 = 3,  %1 = 30,  %2 = 25
❌ 틀림:  %0 = 3,   %0 = 30   (같은 이름 두 번!)
```

### 개념 2: MLIR의 이름표 형태

**형태 1: 번호**
```
%0, %1, %2, %3, ...
```

**형태 2: 의미 있는 이름**
```
%result, %sum, %product, %apple, %juice
```

**규칙**: 모두 % 기호로 시작

### 개념 3: 왜 이 규칙이 필요한가?

**이유 1: 데이터 흐름 추적**
- 어느 값이 어디서 왔는지 명확함

**이유 2: 컴파일러 최적화**
- 죽은 코드 제거 가능
- 계산 미리 하기 가능
- 동시 실행 찾기 가능

**이유 3: 헷갈림 방지**
- %0이 뭔지 명확함
- %1이 뭔지 명확함

## 실습 문제 풀이

### 문제 1: 틀린 그림 찾기

**설계도 A**:
```
%0 = 1 + 2  (결과: 3을 %0에 저장)
%1 = %0 × 10 (결과: 30을 %1에 저장)
```
→ ✅ 올바름 (이름표 다름: %0, %1)

**설계도 B**:
```
%0 = 1 + 2  (결과: 3을 %0에 저장)
%0 = %0 × 10 (결과: 30을 %0에 저장) ← 틀림!
```
→ ❌ 틀림 (이름표 같음: %0, %0)

**나의 답**: 설계도 B가 틀렸다! ✅
