## 핵심 개념 정리

### 개념 1: Dialect (도구 상자)

**정의 (쉬운 말로)**:
- 같은 용도의 도구들을 모아놓은 패키지
- MLIR의 "확장 유닛"
- 상자이름.도구이름 형식으로 사용

**5가지 주요 Dialect**:

| Dialect | 용도 | 도구 예시 |
|---------|------|---------|
| **arith** | 수학 계산 | addi, addf, muli, mulf, divi, divf |
| **func** | 함수 | func, return, call |
| **memref** | 메모리 | alloc, load, store |
| **linalg** | 선형대수 | matmul, dot, transpose |
| **affine** | 루프/제어 | for, if |

### 개념 2: 형식과 마침표의 마법

**형식**:
```
상자이름.도구이름

예시:
arith.addi   ← arith 상자의 addi 도구
func.func    ← func 상자의 func 도구
memref.load  ← memref 상자의 load 도구
```

**마침표(.)의 역할**:
- 상자와 도구를 구분하는 구분자
- "어느 상자의 어느 도구인지" 명확하게 함
- 컴파일러가 올바른 도구를 찾도록 함

### 개념 3: 왜 이렇게 나누어 놓았나?

**이유 1: 명확성**
```
❌ addi %a, %b
   (어디 상자의 addi일까? 불명확!)

✅ arith.addi %a, %b
   (수학 상자의 addi임이 명확!)
```

**이유 2: 확장성**
```
기존 Dialect는 그대로 사용:
- arith.addi (기존 수학)
- func.func (기존 함수)

내가 필요한 것만 추가:
- mydialect.myoperation (새로운 것)

→ 이것이 MLIR의 강점!
```

**이유 3: 재사용성**
```
arith Dialect는 어디서나 사용 가능:
- 함수 안에서
- 루프 안에서
- 메모리 연산 안에서

즉, 도구들을 쉽게 조합할 수 있음
```

### 개념 4: 대학원 예습 - 확장성

**미래의 시나리오**:
```
2030년, 자율주행 자동차 AI 칩 설계 중

❌ 모든 것을 새로 만들기
   시간 낭비, 비효율적

✅ 기존 것 + 내 것 추가
   arith.addi (기존)
   autodrive.detect_lane (새로운)
   → Dialect 확장!
```

**대학원 연구의 흐름**:
```
1학년: 기존 Dialect 배우기
2학년: 자신의 Dialect 설계하기
3학년: 최적화 Pass 만들기

```

## 실습 문제 풀이

### 문제 1: 두 정수를 더하고 싶다면?

**선택지**:
```
A) addi %a, %b : i32
B) arith.addi %a, %b : i32
C) add %a, %b : i32
```

**내 답**: B번 ✅

**이유**:
- 상자: arith (수학 상자)
- 도구: addi (정수 더하기)
- 형식: arith.addi
- 정답 확인됨 ✅

### 문제 2: 두 실수를 더하고 싶다면?

**선택지**:
```
A) arith.add %x, %y : f32
B) arith.addi %x, %y : f32
C) arith.addf %x, %y : f32
```

**내 답**: C번 ✅

**이유**:
- arith.addi는 정수 더하기 (i = integer)
- arith.addf는 실수 더하기 (f = float)
- 우리는 f32(실수)를 다루니까 addf 사용
- 정답 확인됨 ✅

### 문제 3: 함수를 정의하고 싶다면?

**선택지**:
```
A) function.func @add(%a: i32) -> i32
B) func.func @add(%a: i32) -> i32
C) def.func @add(%a: i32) -> i32
```

**내 답**: B번 ✅

**이유**:
- 상자: func (함수 상자)
- 도구: func (함수 정의)
- 형식: func.func
- 정답 확인됨 ✅

### 문제 4: 행렬 곱셈은 어디에서?

**선택지**:
```
A) tensor.matmul %A, %B
B) linalg.matmul %A, %B
C) matrix.matmul %A, %B
```

**내 답**: B번 ✅

**이유**:
- 행렬 연산은 linalg (선형대수 상자)에 있다
- linalg.matmul = 선형대수 상자의 행렬 곱셈
- 정답 확인됨 ✅
