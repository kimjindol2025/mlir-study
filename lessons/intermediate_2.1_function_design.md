# 📦 [중등 2.1] 설계도의 묶음: Function(함수)

> **MLIR의 중등 첫 단계: 개별 명령어를 하나의 기계로 묶기**
>
> "저장 필수. 너는 기록이 증명이다."
>
> 지금까지는 단일 명령어들만 배웠습니다:
> - arith.addi (정수 더하기)
> - arith.mulf (실수 곱하기)
> - memref.load (메모리에서 읽기)
>
> 하지만 실제 프로그램은 이런 명령어들을 한데 모아놓은 꾸러미입니다!
> 이걸 **함수(Function)**라고 부릅니다.

---

## 🎯 오늘 배울 것

단 하나의 개념입니다:

> **"여러 개의 연산은 func.func 안에 모아서 하나의 기계(함수)로 만든다."**
>
> **형식: func.func @이름(%인자: 타입) -> 반환타입 { 본체 }**

## 1️⃣ 상상해 보세요

### 과자를 만드는 기계

상상해 보세요. 자동 과자 만드는 기계가 있습니다.

```
┌─────────────────────────────────┐
│  과자 만드는 기계                  │
│                                 │
│  입력: ┌─────────────────┐     │
│        │ 밀가루 구멍      │     │
│        │ 설탕 구멍        │     │
│        └─────────────────┘     │
│           ↓                    │
│  ┌─────────────────────────┐   │
│  │ 1. 밀가루 + 설탕 섞기   │   │
│  │ 2. 반죽 만들기          │   │
│  │ 3. 170도에서 10분 굽기  │   │
│  │ 4. 식히기               │   │
│  └─────────────────────────┘   │
│           ↓                    │
│  출력: ┌─────────────────┐     │
│        │ 완성된 과자      │     │
│        └─────────────────┘     │
└─────────────────────────────────┘
```

### MLIR의 함수도 똑같습니다!

```
┌─────────────────────────────────┐
│  더하기 기계 (함수)               │
│                                 │
│  입력: ┌─────────────────┐     │
│        │ 첫 번째 수       │     │
│        │ 두 번째 수       │     │
│        └─────────────────┘     │
│           ↓                    │
│  ┌─────────────────────────┐   │
│  │ 두 수를 더하기 (addf)   │   │
│  │ 결과를 저장하기          │   │
│  └─────────────────────────┘   │
│           ↓                    │
│  출력: ┌─────────────────┐     │
│        │ 더한 결과        │     │
│        └─────────────────┘     │
└─────────────────────────────────┘
```

## 2️⃣ 함수의 생김새

### 기본 구조

```mlir
func.func @my_add_machine(%arg0: f32, %arg1: f32) -> f32 {
  // 1. 재료(%arg0, %arg1)를 가져와서 더합니다.
  %result = arith.addf %arg0, %arg1 : f32

  // 2. 완성된 결과물을 기계 밖으로 보냅니다.
  func.return %result : f32
}
```

### 각 부분의 의미

```
func.func        ← func 브랜드의 func 도구 (함수 선언)
@my_add_machine  ← 함수의 이름 (@ 기호 필수!)
(%arg0: f32,    ← 첫 번째 입력 인자 (이름: %arg0, 타입: f32)
 %arg1: f32)    ← 두 번째 입력 인자
-> f32          ← 함수가 반환할 타입 (-> 기호!)
{               ← 함수 본체 시작
  %result = ... ← 함수 안의 명령어들
  func.return   ← 함수에서 나가기
}               ← 함수 본체 끝
```

## 3️⃣ 문법 뜯어보기

### 함수 이름 앞의 @

MLIR에서 이름을 붙일 때:

```
%이름      ← 일반 값 (연산 결과, 임시 변수)
@이름      ← 함수, 전역 변수 (프로그램 전체에서 접근 가능)
```

**규칙**: 함수 이름은 항상 @를 붙인다!

```
✅ @my_add_machine
✅ @calculate_total
✅ @matrix_multiply

❌ %my_add_machine (이건 임시 변수처럼 보임)
❌ my_add_machine (이름이 없는 것처럼 보임)
```

### 인자 (Arguments)

```mlir
func.func @add(%arg0: f32, %arg1: f32) -> f32 {
           ↑     ↑       ↑    ↑       ↑
         함수   인자     타입  인자    타입
         이름   이름          이름
```

**특징**:
- 인자도 %이름 형식 사용
- 각 인자마다 타입 명시
- 여러 개면 쉼표(,)로 구분

### 반환 타입

```mlir
func.func @add(%arg0: f32, %arg1: f32) -> f32 {
                                          ↑
                                      반환 타입
                                      (-> 기호!)
```

**특징**:
- 함수가 돌려줄 값의 타입
- 항상 -> 기호 사용
- 단일 또는 여러 개 가능

## 4️⃣ 실제 예제들

### 예제 1: 정수 더하기 함수

```mlir
func.func @add_integers(%a: i32, %b: i32) -> i32 {
  %result = arith.addi %a, %b : i32
  func.return %result : i32
}
```

**분석**:
- 이름: @add_integers (@ 있음) ✅
- 인자: %a, %b (둘 다 i32) ✅
- 반환: i32 (-> 사용) ✅
- 본체: arith.addi로 더하기 ✅

### 예제 2: 실수 곱하기 함수

```mlir
func.func @multiply_floats(%x: f32, %y: f32) -> f32 {
  %product = arith.mulf %x, %y : f32
  func.return %product : f32
}
```

**분석**:
- 이름: @multiply_floats ✅
- 인자: %x, %y (둘 다 f32) ✅
- 반환: f32 ✅
- 본체: arith.mulf로 곱하기 ✅

### 예제 3: 여러 연산 함수

```mlir
func.func @complex_calculation(%a: i32, %b: i32) -> i32 {
  // 1단계: 더하기
  %sum = arith.addi %a, %b : i32

  // 2단계: 2배하기
  %doubled = arith.muli %sum, 2 : i32

  // 3단계: 10을 더하기
  %const = arith.constant 10 : i32
  %result = arith.addi %doubled, %const : i32

  // 4단계: 반환
  func.return %result : i32
}
```

**분석**:
- 함수 이름: @complex_calculation ✅
- 인자: %a, %b (i32) ✅
- 반환: i32 ✅
- 본체: 여러 명령어를 순서대로 실행 ✅
- 각 단계의 결과를 다음 단계에서 사용 ✅

### 예제 4: 매개변수가 많은 함수

```mlir
func.func @three_way_add(%x: i32, %y: i32, %z: i32) -> i32 {
  %step1 = arith.addi %x, %y : i32
  %final = arith.addi %step1, %z : i32
  func.return %final : i32
}
```

**분석**:
- 3개의 인자를 받음 ✅
- 순서대로 더하기 ✅
- 최종 결과 반환 ✅

## 5️⃣ 올바른 함수 vs 틀린 함수

### ✅ 올바른 함수

```mlir
func.func @add(%a: i32, %b: i32) -> i32 {
  %sum = arith.addi %a, %b : i32
  func.return %sum : i32
}
```

체크리스트:
- ✅ func.func 사용
- ✅ @ 기호 있음
- ✅ 인자 타입 명시
- ✅ -> 기호 있음
- ✅ func.return 사용

### ❌ 틀린 함수 1: @ 기호 생략

```mlir
❌ func.func add(%a: i32, %b: i32) -> i32 {
```

**문제**: 함수 이름 앞에 @ 없음!
**수정**: `func.func @add(...)`

### ❌ 틀린 함수 2: 잘못된 반환 기호

```mlir
❌ func.func @add(%a: i32, %b: i32) : i32 {
                                    ↑
                                  : 아님!
```

**문제**: : 대신 -> 사용해야 함
**수정**: `-> i32`

### ❌ 틀린 함수 3: func.return 없음

```mlir
❌ func.func @add(%a: i32, %b: i32) -> i32 {
  %sum = arith.addi %a, %b : i32
  return %sum : i32  ← func.return이어야!
}
```

**문제**: return이 아니라 func.return 필요
**수정**: `func.return %sum : i32`

### ❌ 틀린 함수 4: 반환 타입 불일치

```mlir
❌ func.func @add(%a: i32, %b: i32) -> f32 {
  %sum = arith.addi %a, %b : i32
  func.return %sum : i32  ← i32를 f32로?
}
```

**문제**: 반환 선언(f32)과 실제 반환값(i32)이 다름
**수정**: `-> i32`로 변경

## 6️⃣ 대학원 수준: 블록(Block)과 영역(Region)

### 개념 이해

함수의 중괄호 { ... } 안쪽 공간을 **영역(Region)**이라고 부릅니다.

```
func.func @add(...) -> ... {
                        ↑ 영역 시작
  (여기 안의 모든 코드)
                        ↓ 영역 끝
}
```

이 영역 안에는 **블록(Block)**이 들어갑니다.

```
func.func @add(...) -> ... {
  // Block 1 (진입 블록)
  %0 = ...
  %1 = ...

  // (나중에 조건문, 루프에서 Block 2, 3 등이 생김)
  // 지금은 Block 1 하나만 있음
}
```

### 지금 알아야 할 것

현재 단계에서는:
```
"함수 안에는 명령어가 순서대로 적히는 공간이 있다"

이 공간을 나중에 분석할 때:
- Region = 함수 내부의 모든 공간
- Block = 순차적으로 실행되는 명령어들의 모음
```

### 나중에 배울 것 (대학원)

복잡한 제어흐름(루프, 조건문)이 나올 때:
```
func.func @conditional(...) {
  // Block 1: 조건 확인

  // Block 2: True인 경우
  // Block 3: False인 경우

  // Block 4: 합쳐진 후
}
```

## 7️⃣ 중등 2.1의 핵심 정리

### 함수의 기본 형식

```
func.func @함수이름(%인자1: 타입1, %인자2: 타입2, ...) -> 반환타입 {
  // 함수 본체
  // 여러 개의 연산들...
  func.return %결과 : 반환타입
}
```

### 체크리스트

함수를 작성할 때 다음을 확인하세요:

- [ ] func.func 사용했나요?
- [ ] @ 기호를 함수 이름 앞에 붙였나요?
- [ ] 모든 인자에 타입을 명시했나요?
- [ ] -> 기호를 사용해서 반환 타입을 명시했나요?
- [ ] func.return을 사용해서 값을 반환했나요?
- [ ] 반환하는 값의 타입이 선언한 타입과 일치하나요?

## 9️⃣ 중등 2.1 기록 (증명)

> **"여러 개의 연산은 func.func 안에 모아서 하나의 기계(함수)로 만든다."**
>
> **형식**: `func.func @이름(%인자: 타입) -> 반환타입 { ... }`
>
> **핵심 기호**:
> - @ (함수 이름 앞)
> - -> (반환 타입 앞)
> - func.return (함수 반환)
>
> 이제 당신은 **프로그래밍의 가장 기본 단위인 "함수"**를 이해했습니다!
> 이것이 모든 소프트웨어의 기초입니다.
