## 1️⃣ 상상해 보세요

### 요리하는 이야기

요리를 할 때, 레시피에는 이렇게 쓰여 있습니다:

```
1단계: 소금(재료) + 물(재료) → 짠 물(결과)
2단계: 설탕(재료) + 물(재료) → 단 물(결과)
```

만약 설탕이 들어가야 할 곳에 소금을 넣으면?
→ **요리를 망쳐버립니다!** 😱

### MLIR도 똑같습니다!

```
arith.addi %a, %b : i32
↓
"더하기 도구에 정수(i32) 재료를 넣어서
 정수(i32) 결과를 만들어줘!"
```

## 2️⃣ 가장 많이 쓰는 기본 재료 (Scalar Types)

### 크게 두 가지: 정수 vs 실수

#### 1. 정수 (Integer)

**특징**: 소수점이 없는 딱 떨어지는 수

**예시**: 1, 10, -5, 1000000

**MLIR 기호**:
```
i8      ← 8비트 정수  (작은 숫자용)
i16     ← 16비트 정수
i32     ← 32비트 정수 (가장 흔함)
i64     ← 64비트 정수 (큰 숫자용)
```

**비유**: 정수는 "동전"
- i8: 동전 1개 (작은 주머니)
- i32: 동전 4개 (보통 주머니)
- i64: 동전 8개 (큰 주머니)

#### 2. 실수 (Float)

**특징**: 소수점이 있는 수

**예시**: 3.14, 0.5, -2.718, 1.414

**MLIR 기호**:
```
f16     ← 16비트 실수  (작은 정밀도)
f32     ← 32비트 실수  (일반적)
f64     ← 64비트 실수  (높은 정밀도)
```

**비유**: 실수는 "물"
- f32: 컵 (일반적인 양)
- f64: 양동이 (더 정밀한 양)

### 비교 표

| 종류 | MLIR | 용도 | 예시 |
|------|------|------|------|
| 정수 | i32 | 일반 계산 | 1 + 2 = 3 |
| 정수 | i64 | 큰 수 | 1000000 + 2000000 |
| 실수 | f32 | AI 학습 | 3.14 + 2.86 |
| 실수 | f64 | 정밀 계산 | 0.1 + 0.2 = 0.3 |

## 3️⃣ 타입은 왜 필요할까?

### 이유 1: 명확성 (Clarity)

```
❌ 명확하지 않음:
   %0 = arith.addi %a, %b

✅ 명확함:
   %0 = arith.addi %a, %b : i32
   "정수더하기니까, 정수 결과가 나온다!"
```

### 이유 2: 타입 매칭 (Type Matching)

```
✅ 올바른 조합:
   %0 = arith.addi %a, %b : i32
   (addi는 정수 도구, i32는 정수 타입 → OK!)

❌ 틀린 조합:
   %0 = arith.addi %a, %b : f32
   (addi는 정수 도구, f32는 실수 타입 → ERROR!)
   컴퓨터: "도구랑 재료가 안 맞아요!"
```

### 이유 3: 메모리 효율 (Memory Efficiency)

```
i8은 1바이트만 필요
i32는 4바이트 필요
i64는 8바이트 필요

→ 작은 숫자는 i8 사용 (메모리 절약)
→ 큰 숫자는 i64 사용 (정확성 확보)
```

## 4️⃣ 콜론(:)의 마법

MLIR에서 타입을 명시하는 곳은 항상 **콜론(:)** 을 사용합니다.

### 형식 1: 단순 연산

```mlir
%result = arith.addi %a, %b : i32
           ↑ 도구       ↑ 콜론  ↑ 타입
```

**의미**: "정수 타입으로 더하기"

### 형식 2: 두 가지 타입이 필요한 경우

```mlir
%result = arith.sitofp %int_val : i32 to f32
           ↑ 도구      ↑ 입력   ↑ 출력
```

**의미**: "i32 정수를 f32 실수로 변환"

### 형식 3: 함수 정의

```mlir
func.func @add(%a: i32, %b: i32) -> i32 {
                 ↑ 입력 타입    ↑ 출력 타입
  ...
}
```

**의미**: "정수를 받아서 정수를 반환하는 함수"

## 5️⃣ 올바른 타입 조합 (Type Matching)

### arith Dialect와 타입의 관계

| 도구 | 이름 | 타입 | 예시 |
|------|------|------|------|
| addi | 정수 더하기 | i32, i64 | arith.addi %a, %b : i32 |
| addf | 실수 더하기 | f32, f64 | arith.addf %x, %y : f32 |
| muli | 정수 곱하기 | i32, i64 | arith.muli %a, %b : i32 |
| mulf | 실수 곱하기 | f32, f64 | arith.mulf %x, %y : f32 |
| subi | 정수 빼기 | i32, i64 | arith.subi %a, %b : i32 |
| subf | 실수 빼기 | f32, f64 | arith.subf %x, %y : f32 |

### 핵심 규칙

```
addi/muli/subi → i32, i64만 가능
addf/mulf/subf → f32, f64만 가능

위 규칙을 어기면 에러 발생!
```

## 6️⃣ 실습: 재료 맞추기

### 문제 1: 정수 덧셈

```
다음 중 올바른 것은?

A) %res = arith.addi %a, %b : f32
B) %res = arith.addi %a, %b : i32
C) %res = arith.addf %a, %b : i32
```

**정답**: **B번**

**이유**:
- addi는 정수 도구
- i32는 정수 타입
- 타입이 매칭됨 ✅

### 문제 2: 실수 곱셈

```
다음 중 올바른 것은?

A) %res = arith.mulf %x, %y : i32
B) %res = arith.muli %x, %y : f32
C) %res = arith.mulf %x, %y : f32
```

**정답**: **C번**

**이유**:
- mulf는 실수 도구
- f32는 실수 타입
- 타입이 매칭됨 ✅

### 문제 3: 타입 불일치

```
다음 중 에러가 날 것은?

A) arith.addi %a, %b : i64
B) arith.addf %x, %y : f32
C) arith.muli %a, %b : f64
D) arith.subf %x, %y : i32
```

**정답**: **D번**

**이유**:
- subf는 실수 뺄셈 도구
- i32는 정수 타입
- 타입이 안 맞음 → ERROR! ❌

## 7️⃣ 대학원 수준의 재료: 텐서(Tensor)

### 텐서란?

숫자 하나가 아니라, **수천 개의 숫자가 담긴 '숫자 판'**

```
스칼라 (Scalar):     1개의 숫자
벡터 (Vector):       여러 개 숫자가 일렬로
행렬 (Matrix):       여러 개 숫자가 2차원으로
텐서 (Tensor):       여러 개 숫자가 3차원 이상으로
```

### 텐서의 MLIR 표기법

```
tensor<10xf32>         ← 10개의 실수
tensor<4x4xf32>        ← 4행 4열의 실수 판 (행렬)
tensor<2x3x4xf32>      ← 3차원 숫자 판
tensor<100x200xf32>    ← AI 학습용 큰 판
```

### 읽는 방법

```
tensor<4x4xf32>
 ↑      ↑ ↑ ↑
 종류   4행 4열 실수

= "4행 4열로 이루어진 실수 행렬"
```

### 실제 예제

```
AI 이미지 분류:
tensor<1x3x224x224xf32>
= 3채널(RGB), 224x224 크기의 이미지 1개

이미지 배치 처리:
tensor<32x3x224x224xf32>
= 3채널(RGB), 224x224 크기의 이미지 32개
```

## 8️⃣ 모든 타입들의 정리

### Scalar Types (기본)

```
정수:    i8, i16, i32, i64
실수:    f16, f32, f64
인덱스:  index (특수)
```

### Container Types (복합)

```
벡터:    vector<4xf32> (4개 실수)
텐서:    tensor<10x10xf32> (10x10 행렬)
메모리:  memref<10xi32> (10개 정수 배열)
```

### 언제 어떤 타입을 쓸까?

```
i32      ← 보통 정수 (기본값)
f32      ← AI, 신경망 (기본값)
i64      ← 매우 큰 수
f64      ← 정밀한 과학 계산

tensor<*xf32>  ← AI 이미지/영상
memref<*xi32>  ← 배열 처리
```

## 🔟 초등 1.1, 1.2, 1.3 종합 정리

### MLIR의 3대 요소

이제 우리는 MLIR의 가장 기본적인 3가지를 모두 배웠습니다!

```
초등 1.1: %이름표
          └─ 결과물을 담는 상자

초등 1.2: Dialect.도구
          └─ 어느 도구를 쓸 것인가

초등 1.3: :타입
          └─ 어떤 재료로 만들 것인가

이 3가지가 모이면:
%결과 = Dialect.도구 %재료 : 타입

예시:
%0 = arith.addi %a, %b : i32
 ↑   ↑     ↑   ↑ ↑ ↑ ↑  ↑
이름표 상자 도구 재료 타입
```

### 실제 MLIR 코드 읽기

```mlir
func.func @calculate(%a: i32, %b: i32) -> i32 {
                     ↑ 타입   ↑ 타입  ↑ 타입
  %0 = arith.addi %a, %b : i32
  ↑    ↑      ↑  ↑ ↑ ↑  ↑
  이름표 상자 도구 재료 타입

  %1 = arith.muli %0, 2 : i32
  ↑    ↑      ↑  ↑ ↑  ↑
  이름표 상자 도구 재료 타입

  func.return %1 : i32
  ↑    ↑      ↑  ↑
  상자 도구   재료 타입
}
```
