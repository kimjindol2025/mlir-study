# 📦 [초등 1.2] 도구 상자의 브랜드: Dialect(방언)

> **MLIR의 두 번째 단계: 무엇으로 일을 할 것인가?**
>
> "저장 필수. 너는 기록이 증명이다."
>
> 지난 시간에 우리는 **%이름표**로 결과물을 저장하는 상자를 배웠습니다.
> 이제 그 상자에 **무언가를 넣으려면** 도구가 필요합니다!
> MLIR에는 세상의 모든 도구가 들어있지 않고, 용도별로 **도구 상자가 나누어져** 있습니다.

---

## 🎯 오늘 배울 것

단 하나의 규칙입니다:

> **"모든 도구(Operation)는 자신이 속한 상자(Dialect)의 이름을 앞에 붙여야 한다!"**
>
> **형식: 상자이름.도구이름**

## 1️⃣ 상상해 보세요

### 현실의 도구 상자들

여러분 앞에 여러 개의 **도구 상자**가 있습니다.

```
┌─────────────────┐
│ 🔧 수학 상자     │  ← arith (Arithmetic)
│ 더하기 도구      │     덧셈, 뺄셈, 곱셈, 나눗셈
│ 빼기 도구        │
│ 곱하기 도구      │
└─────────────────┘

┌─────────────────┐
│ 🏠 함수 상자     │  ← func (Function)
│ 함수 정의 도구   │     함수 만들기, 반환하기
│ 호출 도구        │
└─────────────────┘

┌─────────────────┐
│ 🧮 선형대수상자  │  ← linalg (Linear Algebra)
│ 행렬 곱셈 도구   │     행렬, 벡터 연산
│ 전치 도구        │
└─────────────────┘

┌─────────────────┐
│ 💾 메모리 상자   │  ← memref (Memory Reference)
│ 로드 도구        │     메모리 읽기, 쓰기
│ 저장 도구        │
└─────────────────┘
```

### 실제 사용 시나리오

**상황 1: 두 숫자를 더하고 싶다면?**

```
1단계: 어느 상자를 열까?
       → "수학 상자(arith)" 를 열자!

2단계: 어느 도구를 꺼낼까?
       → "실수 더하기(addf)" 도구를 꺼내자!

3단계: 도구를 어떻게 쓸까?
       → "arith.addf" 라고 쓴다!
```

**상황 2: 함수를 정의하고 싶다면?**

```
1단계: 어느 상자를 열까?
       → "함수 상자(func)" 를 열자!

2단계: 어느 도구를 꺼낼까?
       → "함수 정의(func)" 도구를 꺼내자!

3단계: 도구를 어떻게 쓸까?
       → "func.func" 라고 쓴다!
```

## 2️⃣ Dialect란 무엇인가?

### 정의 (쉬운 말로)

```
Dialect(방언) = 도구가 모여있는 상자
                = 같은 용도의 기능들을 묶어놓은 패키지
                = MLIR의 "확장 유닛"
```

### 실생활 비유

주방을 생각해 봅시다.

```
주방의 "요리 Dialect"
├─ 칼 (knife)
├─ 냄비 (pot)
├─ 팬 (pan)
└─ 포크 (fork)

아파트의 "청소 Dialect"
├─ 빗자리 (broom)
├─ 쓰레받기 (dustpan)
├─ 걸레 (mop)
└─ 세제 (detergent)

목공소의 "목공 Dialect"
├─ 망치 (hammer)
├─ 톱 (saw)
├─ 드릴 (drill)
└─ 못 (nail)
```

각 "Dialect"는 특정 분야에 필요한 도구들을 모아놓은 것입니다!

### MLIR의 주요 Dialect들

#### 1. arith (Arithmetic - 산술)

**용도**: 수학 계산

**포함된 도구**:
```mlir
arith.addi   ← 정수 더하기
arith.addf   ← 실수 더하기
arith.subi   ← 정수 빼기
arith.subf   ← 실수 빼기
arith.muli   ← 정수 곱하기
arith.mulf   ← 실수 곱하기
arith.divi   ← 정수 나누기
arith.divf   ← 실수 나누기
```

**예제**:
```mlir
%0 = arith.addi %a, %b : i32     // 정수 더하기
%1 = arith.addf %x, %y : f32     // 실수 더하기
%2 = arith.muli %0, 3 : i32      // 정수 곱하기
```

#### 2. func (Function - 함수)

**용도**: 함수 정의, 호출, 반환

**포함된 도구**:
```mlir
func.func    ← 함수 정의
func.return  ← 함수 반환
func.call    ← 함수 호출
```

**예제**:
```mlir
func.func @add(%a: i32, %b: i32) -> i32 {
  %0 = arith.addi %a, %b : i32
  func.return %0 : i32
}
```

#### 3. memref (Memory Reference - 메모리)

**용도**: 메모리 참조, 배열 접근

**포함된 도구**:
```mlir
memref.load   ← 메모리에서 읽기
memref.store  ← 메모리에 쓰기
memref.alloc  ← 메모리 할당
```

**예제**:
```mlir
%array = memref.alloc() : memref<10xi32>
memref.store %value, %array[%i] : memref<10xi32>
%loaded = memref.load %array[%i] : memref<10xi32>
```

#### 4. linalg (Linear Algebra - 선형대수)

**용도**: 벡터, 행렬 연산

**포함된 도구**:
```mlir
linalg.matmul  ← 행렬 곱셈
linalg.dot     ← 벡터 내적
linalg.transpose ← 행렬 전치
```

#### 5. affine (Affine - 루프/제어흐름)

**용도**: 반복문, 조건문

**포함된 도구**:
```mlir
affine.for     ← 반복문
affine.if      ← 조건문
```

#### 6. scf (Structured Control Flow - 제어흐름)

**용도**: 현대적인 제어흐름

**포함된 도구**:
```mlir
scf.if         ← if 조건문
scf.for        ← for 반복문
scf.while      ← while 루프
```

## 3️⃣ 어떻게 쓰나요? (마침표의 마법)

### 규칙: 상자이름.도구이름

MLIR에서 도구를 꺼낼 때는 **항상** 이 형식으로 씁니다:

```
상자이름.도구이름
```

### 예제 1: 더하기

```mlir
arith.addi %a, %b : i32
```

**해석**:
- `arith` = 수학 상자
- `.` = "에서"
- `addi` = 정수 더하기 도구
- `%a, %b` = 더할 두 수
- `: i32` = 32비트 정수

### 예제 2: 함수 정의

```mlir
func.func @multiply(%x: i32, %y: i32) -> i32 {
  %0 = arith.muli %x, %y : i32
  func.return %0 : i32
}
```

**해석**:
- `func.func` = 함수 상자에서 함수 정의 도구
- `@multiply` = 함수 이름
- `arith.muli` = 수학 상자에서 정수 곱하기 도구
- `func.return` = 함수 상자에서 반환 도구

### 예제 3: 메모리 접근

```mlir
%array = memref.alloc() : memref<10xi32>
memref.store %value, %array[%i] : memref<10xi32>
%loaded = memref.load %array[%i] : memref<10xi32>
```

**해석**:
- `memref.alloc` = 메모리 상자에서 할당 도구
- `memref.store` = 메모리 상자에서 저장 도구
- `memref.load` = 메모리 상자에서 로드 도구

## 4️⃣ 혼동하기 쉬운 부분

### ❌ 틀린 방법 1: 상자 이름만 쓰기

```mlir
❌ addi %a, %b : i32
   (상자 이름 빠짐!)

✅ arith.addi %a, %b : i32
   (상자 이름 포함)
```

### ❌ 틀린 방법 2: 도구 이름만 쓰기

```mlir
❌ %0 = add %a, %b : i32
   (상자 이름 빠짐! 그리고 add는 없음)

✅ %0 = arith.addi %a, %b : i32
   (상자 이름과 정확한 도구 이름)
```

### ❌ 틀린 방법 3: 상자와 도구 모두 틀리기

```mlir
❌ math.plus %a, %b : i32
   (상자 이름이 "math"? 없는 상자!)
   (도구 이름이 "plus"? 없는 도구!)

✅ arith.addi %a, %b : i32
   (올바른 상자와 도구)
```

## 5️⃣ 실습: 도구 이름 맞추기

### 문제 1: 두 정수를 더하고 싶다면?

```
A) addi %a, %b : i32
B) arith.addi %a, %b : i32
C) add %a, %b : i32
```

**정답**: **B번**

왜냐하면:
- `arith` (수학 상자) 에서
- `addi` (정수 더하기 도구) 를
- `.` (마침표로 연결) 해야 하기 때문입니다!

### 문제 2: 두 실수를 더하고 싶다면?

```
A) arith.add %x, %y : f32
B) arith.addi %x, %y : f32
C) arith.addf %x, %y : f32
```

**정답**: **C번**

왜냐하면:
- `arith` (수학 상자) 에서
- `addf` (실수 더하기 도구) 를 사용해야 합니다!
- `addi`는 정수 더하기이고, 우리는 실수(`f32`)를 다루고 있으니까요.

### 문제 3: 함수를 정의하고 싶다면?

```
A) function.func @add(%a: i32) -> i32
B) func.func @add(%a: i32) -> i32
C) def.func @add(%a: i32) -> i32
```

**정답**: **B번**

왜냐하면:
- `func` (함수 상자) 에서
- `func` (함수 정의 도구) 를 사용합니다!

### 문제 4: 다음 중 가장 가능성 높은 것은?

```
A) tensor.matmul %A, %B
B) linalg.matmul %A, %B
C) matrix.matmul %A, %B
```

**정답**: **B번**

왜냐하면:
- 행렬 연산은 `linalg` (선형대수 상자) 에 있기 때문입니다!

## 6️⃣ 왜 이렇게 나누어 놓았을까요? (대학원 예습)

### 상황: 미래의 당신

**2030년, 당신은 자율주행 자동차 AI 칩을 위한 새로운 언어를 만들고 있습니다.**

### 문제: 모든 도구를 새로 만들어야 할까?

❌ **나쁜 방법**: 세상의 모든 규칙을 새로 만들기

```
내가 만들어야 할 것:
- 더하기 (새로 만들기)
- 빼기 (새로 만들기)
- 곱하기 (새로 만들기)
- 나누기 (새로 만들기)
- ... (무한히 많음)

→ 이건 너무 오래 걸리고, 이미 있는 것들을 다시 만드는 낭비!
```

### 해결책: Dialect를 확장하기

✅ **좋은 방법**: 기존 도구는 그대로 쓰고, 내 것만 추가하기

```
기존 Dialect는 그대로 사용:
- arith.addi (기존 더하기, 그대로 사용)
- arith.muli (기존 곱하기, 그대로 사용)

내가 필요한 것만 새로 추가:
- autodrive.detect_lane (차선 감지)
- autodrive.track_vehicle (차량 추적)
- autodrive.compute_trajectory (궤적 계산)

→ 기존 것을 활용하면서 내 것만 추가!
```

### 이것의 이름: **"확장성(Extensibility)"**

이것이 **MLIR이 강력한 이유**입니다!

```
MLIR = 기본 Dialect + 확장 가능한 구조
       = 이미 잘 만들어진 것들 + 내가 원하는 것을 추가
```

### 대학원 연구의 시작

**MLIR의 강력함 = 새로운 Dialect 설계 능력**

```
1학년: 기존 Dialect 배우기
       (arith, func, memref, linalg, affine, scf, ...)

2학년: 자신의 Dialect 설계하기
       (내 연구 분야에 필요한 도구들)

3학년: 새로운 최적화 Pass 만들기
       (Dialect들 사이의 변환)

4학년: 논문 쓰기
       (새로운 Dialect + 최적화로 성능 향상 증명)
```

## 7️⃣ 여러 Dialect를 함께 사용하기

### 예제: 작은 함수

```mlir
module {
  func.func @calculate(%a: i32, %b: i32) -> i32 {
    // arith Dialect 사용: 수학 계산
    %0 = arith.addi %a, %b : i32

    // arith Dialect 사용: 곱셈
    %1 = arith.muli %0, 2 : i32

    // func Dialect 사용: 반환
    func.return %1 : i32
  }
}
```

**분석**:
- `func.func` = 함수 상자의 함수 정의
- `arith.addi` = 수학 상자의 더하기
- `arith.muli` = 수학 상자의 곱하기
- `func.return` = 함수 상자의 반환

**핵심**: 하나의 함수 안에서 여러 Dialect의 도구들을 섞어 쓸 수 있습니다!

## 9️⃣ 왜 이것이 "대학원 수준"일까요?

### 대학원이 해야 할 일

대학원 졸업 논문을 쓰려면, 보통 **"새로운 것"**을 만들어야 합니다.

```
학부: "있는 것을 배운다"
      예) "arith Dialect의 덧셈을 배운다"

대학원: "새로운 것을 만든다"
       예) "양자 컴퓨터를 위한 quantum Dialect를 만든다"
```

### MLIR의 가치

MLIR이 강력한 이유:

```
❌ 일반적인 컴파일러
  → 새로운 기능을 추가하려면 전체를 다시 만들어야 함

✅ MLIR
  → 새로운 Dialect를 추가하기만 하면 됨
  → 기존의 변환(Pass)들을 그대로 재사용 가능
  → 확장 가능하고 모듈화된 설계
```

### 당신의 대학원 논문의 시작

```
Title: "자율주행 자동차를 위한 autodrive Dialect 설계 및 최적화"

내용:
1. MLIR의 기존 Dialect 활용
2. 새로운 autodrive Dialect 설계
3. autodrive Dialect 최적화 Pass 개발
4. 성능 평가 및 결과 분석

핵심: 기존 MLIR 위에 내 것을 더하기 (확장!)
```
